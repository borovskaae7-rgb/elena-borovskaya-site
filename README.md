# AI-аудит сообществ ВКонтакте

В репозитории есть исходный лендинг и готовое Python-приложение `vk_checker` для массовой проверки сообществ ВКонтакте из Google Таблицы на признаки инфобизнеса, онлайн-образования, экспертности и продажи образовательных продуктов.

## Возможности `vk_checker`

- Читает ссылки на VK-сообщества из первого столбца Google Таблицы.
- Открывает страницы через Playwright Chromium с persistent context, чтобы авторизация ВКонтакте сохранялась между запусками.
- Собирает название, описание, статус, адрес, меню, ссылки, посты, услуги, товары, контакты, кнопки, вид деятельности и подписчиков.
- Передаёт собранные данные в OpenAI API и получает строгий JSON с классификацией.
- Записывает обратно в таблицу столбцы: `Инфобизнес`, `Категория`, `Подкатегория`, `Уверенность`, `Причина`, `Последняя проверка`.
- После остановки продолжает с первой строки, где не заполнена `Последняя проверка`.
- Поддерживает параллельную обработку, ретраи сетевых запросов, rate limit OpenAI и подробные логи.

## Требования

- Python 3.12
- Доступ к Google Cloud проекту с Google Sheets API
- Service Account JSON с доступом к таблице
- OpenAI API key
- Доступ к ВКонтакте в браузере Playwright

## Установка

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
cp .env.example .env
```

Заполните `.env`:

```env
OPENAI_API_KEY=sk-your-key
GOOGLE_SHEET_ID=your-google-sheet-id
GOOGLE_SERVICE_ACCOUNT=./service-account.json
HEADLESS=false
MAX_CONCURRENT=2
MODEL=gpt-4.1-mini
VK_STORAGE_STATE=.vk-profile
INPUT_SHEET_NAME=Лист1
INPUT_COLUMN=A
START_ROW=2
OPENAI_RPM=60
REQUEST_TIMEOUT=60
LOG_LEVEL=INFO
```

`GOOGLE_SERVICE_ACCOUNT` может быть путём к JSON-файлу или JSON-строкой service account.

## Как создать Google API и подключить таблицу

1. Откройте [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте проект или выберите существующий.
3. Включите **Google Sheets API**.
4. Создайте **Service Account**.
5. Создайте JSON-ключ и сохраните его, например, как `service-account.json` в корне проекта.
6. Откройте Google Таблицу и нажмите **Поделиться**.
7. Добавьте email service account из JSON-файла, например `name@project.iam.gserviceaccount.com`, с правами редактора.
8. Скопируйте ID таблицы из URL и укажите его в `GOOGLE_SHEET_ID`.

## Формат таблицы

- Первый столбец (`A`) содержит ссылки:
  - `https://vk.com/club12345`
  - `https://vk.com/public12345`
  - `https://vk.com/example`
- Первая строка — заголовки.
- Данные начинаются со строки `START_ROW`, по умолчанию со 2-й.
- Приложение само добавит недостающие столбцы результата в первую строку.

## Как авторизоваться во ВКонтакте

Первый запуск рекомендуется делать с `HEADLESS=false`:

```bash
source .venv/bin/activate
python -m vk_checker
```

Откроется Chromium Playwright. Если VK попросит логин, авторизуйтесь вручную. Профиль браузера сохранится в папке из `VK_STORAGE_STATE`, по умолчанию `.vk-profile`. Следующие запуски будут использовать ту же сессию. После успешной авторизации можно поставить `HEADLESS=true`, если аккаунт не требует дополнительной проверки.

## Запуск проверки

```bash
source .venv/bin/activate
python -m vk_checker
```

Логи пишутся в:

- `logs/app.log` — общий лог;
- `logs/errors.log` — ошибки.

## Продолжение после остановки

Ничего дополнительно делать не нужно. При следующем запуске приложение прочитает таблицу и обработает только строки, где столбец `Последняя проверка` пустой. Уже обработанные сообщества повторно не проверяются.

## Как изменить количество потоков

Параллельность задаётся переменной:

```env
MAX_CONCURRENT=4
```

Увеличивайте значение постепенно. Для больших таблиц на 10 000+ строк учитывайте ограничения VK, Google Sheets API и OpenAI API. Если OpenAI начинает отвечать ошибками лимитов, уменьшите `MAX_CONCURRENT` или `OPENAI_RPM`.

## Модель OpenAI и лимиты

Модель задаётся переменной:

```env
MODEL=gpt-4.1-mini
```

Ограничение запросов OpenAI в минуту задаётся так:

```env
OPENAI_RPM=60
```

Классификатор использует `response_format={"type":"json_object"}` и валидирует ответ через Pydantic.

## Категории классификации

Приложение использует категории из задания: онлайн-школа, эксперт, коуч, наставник, психолог, продюсер, маркетинг, SMM, таргет, маркетплейсы, инвестиции, финансы, недвижимость, языковая школа, детское образование, подготовка к ЕГЭ, бизнес, фриланс, дизайн, IT, программирование, нутрициология, фитнес, красота, медицина, другое и не относится.
